

# CI/CD Basics for DevOps

This repository explains Continuous Integration and Continuous Deployment from a beginner’s perspective.

## What is CI?
Continuous Integration means automatically testing and validating code whenever changes are pushed.

## What is CD?
Continuous Deployment/Delivery means automatically deploying tested code to servers.

## Why CI/CD is Important
- Reduces manual work
- Catches bugs early
- Improves release speed
- Increases reliability

## Simple CI/CD Flow
Code → GitHub → CI tests → Build → Deploy

## Tools Commonly Used
- GitHub Actions
- Jenkins
- GitLab CI
- AWS services

## Learning Goal
Understand the CI/CD mindset before building pipelines.
